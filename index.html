Introduction to Github analysis

WHAT IS IT

About Project

The main idea of our project is to use Github to visualize connection between users and public repositories present on their website. It is interesting because there is a vast amount of data on the github regarding to connections of the users to the repositories, what they are contributing and what they like and then they can share with each other and so on.

Aim of this project is to show on graphs relations between repositories and users and show some intresting statistics

WHAT DATA

Data collection

Github rest api requests are used to get all the data we need(if you want to see solution go to our github page). All the data was acquired as Json and put into the SQLite database.
Github has over 36 millions repositories and that translates over 20 GB data. That is a lots of data and gets over 360, 000 requests that takes over 72 hours (we have a limit to get 30 requests per minute of search and 5000 per hour of other commands).

HOW IS IT DONE

Solution

Data is limited by query "game" (e.g. Repository for game development, the games created by users, and so on) it includes around 849.000 repositories that translates around 5 GB data. It gets 8500 requests that takes almost 2 hours just for the lists. We also have to take each profile and each contributor for the repository. It takes time, more than 2 hours but the geting of data is not so much time consuming like the Github repository case. We also did a request for "social graphs". It has just 787 repositories so it does not include enough data.
